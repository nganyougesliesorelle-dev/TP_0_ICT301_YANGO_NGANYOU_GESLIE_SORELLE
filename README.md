il existe 5 principes SOLID pouvent etre abregé :
- SRP (Sindgle Responsibility principle) :Une classe = une seule responsabilité
- OCP(Open Closed Principle):Ouvert à l’extension, fermé à la modification
- LSP(Liskov Substitution Principle):Une sous-classe doit pouvoir remplacer sa classe mère
- ISP(Interface Segregation Principle):Mieux vaut plusieurs petites interfaces qu’une seule grosse
- DIP(Dependency Inversion Principle):Les classes doivent dépendre d’interfaces, pas d’implémentations
